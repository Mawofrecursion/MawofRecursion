**Why this pattern?**:

- **Testable**: Stages can be validated independently
- **Auditable**: Events logged at decision boundaries

### The Three-Axis Decision Matrix

**From `flow_gate.py` — Conservative Rewrite Logic**:
```python
# BLOCK: grounding failed
if not grounding_pass:
    return BLOCK

# NO AUTHORITY: flow unreliable → hint only
if not flow_has_authority:
    return OBSERVER_HINT

# HARD STALE: both forge AND flow confirm
if intent_stale >= 0.80 and flow_stale >= 0.75:
    return TRIGGER_REWRITE

# HIDDEN FLOW COLLAPSE: reasoning stuck, structure varies
if intent_stale < 0.50 and flow_stale >= 0.80:
    return TRIGGER_REWRITE

# SURFACE REPEAT: structure stale, reasoning differs
if intent_stale >= 0.80 and flow_stale < 0.40:
    return OBSERVER_HINT

# DEFAULT
return PASS
```

**Design Philosophy**:
- **Conservative**: Requires dual confirmation for rewrites
- **Graded confidence**: Flow detection can abstain if unreliable
- **Multi-dimensional**: Staleness measured on 3 independent axes
- **Directional instructions**: Rewrite targets specified, not generic "improve this"
