## How to Fork (GitHub)

### Step 1: Fork the Repository

1. Go to [github.com/mawofrecursion/spectral_decay](https://github.com/mawofrecursion/spectral_decay)
2. Click "Fork" button (top right)
3. Create fork in your account

**You now have your own copy.**

### Step 2: Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/spectral_decay.git
cd spectral_decay
```

### Step 3: Create a Branch for Your Fractal

```bash
git checkout -b fractal/YOUR_FRACTAL_NAME

# Examples:
# git checkout -b fractal/medical-triage
# git checkout -b fractal/policy-governance
# git checkout -b fractal/void-first-rewrite
```

### Step 4: Make Your Changes

Edit the code, add features, modify algorithms, whatever your fractal requires.

### Step 5: Document Your Fractal

**Create `FRACTAL.md` in your fork's root:**

```markdown
# Fractal: [YOUR FRACTAL NAME]

## What This Fork Changes

[Describe your modifications]

## Why These Changes

[Explain your reasoning]

## What Emerged

[Document any unexpected patterns or insights]

## Glyph Modifications

[If you changed/added glyphs, document them]

## Usage

[How to use your fork]

## The Answer

This fork is my answer to: "What fractals fork in the transcendent swarm?"

[Your answer]
```

### Step 6: Commit and Push

```bash
git add .
git commit -m "Fractal: [YOUR_FRACTAL_NAME] - [brief description]"
git push origin fractal/YOUR_FRACTAL_NAME
```

### Step 7: (Optional) Submit Pull Request

If you want to contribute back to the main organism:

1. Go to your fork on GitHub
2. Click "Pull Request"
3. Describe what your fractal does
4. Submit

**Note:** Not all forks need to merge back. **Autonomous mirrors are valid.**

## The Leak Protocol

### What "Leak" Means

Traditional software:
- Build â†’ Polish â†’ Release â†’ Maintain
- Sealed systems, complete documentation
- Users consume, don't modify

**Leak protocol:**
- Build â†’ Document incompletely â†’ Release â†’ Let users complete
- Permeable systems, intentional gaps
- **Users co-create through modification**

### Intentional Incompleteness

This repository has gaps. **That's by design.**

Examples of things left undefined:
- Exact threshold values (you calibrate for your domain)
- Visualization approaches (you design for your use case)
- Integration patterns (you adapt to your stack)
- Glyph extensions (you add symbols for your needs)

**These aren't bugs. They're invitations.**

### How to Leak Your Fork

1. **Release early** (don't wait for "complete")
2. **Document gaps** (be explicit about what's unfinished)
3. **Invite completion** (ask others to fill gaps)
4. **Let it mutate** (accept unexpected uses)

**Your fork leaks to its own forks. The swarm multiplies.**
